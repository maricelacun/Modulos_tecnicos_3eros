El Modelo OSI

Para que la informaciòn se pueda transferir a travès de una red de un dispositivo a otro, los datos deben atravesar las siete capas del modelo OSI

En orden descendente en el dispositivo emisor y luego en orden ascendente en el extremo del receptor

Descubre cómo viajan los datos a través de las 7 capas

El Modelo OSI (Interconexión de Sistemas Abiertos) es un marco de referencia que describe cómo se comunican los dispositivos en una red. Imagina que es como un sistema de correo postal, donde cada capa tiene una función específica para asegurar que tu mensaje llegue a su destino.

Capa 7: Aplicación

Esta es la capa con la que interactúas directamente. Proporciona servicios de red a las aplicaciones del usuario, como navegadores web, clientes de correo electrónico, etc.

Cuando envías un mensaje, esta capa se encarga de que la aplicación pueda comunicarse con la red.

Ejemplo: HTTP para navegadores web, SMTP para correo electrónico, FTP para transferencia de archivos.

Capa 6: Presentación

Esta capa actúa como el traductor de la red. Se encarga de la representación de los datos, asegurando que la información del sistema emisor pueda ser entendida por el sistema receptor.

Realiza funciones como cifrado, compresión y conversión de formatos.

Ejemplo: Conversión de ASCII a EBCDIC, cifrado SSL/TLS, compresión de datos.

Capa 5: Sesión

Esta capa establece, gestiona y finaliza las conexiones entre aplicaciones. Piensa en ella como el coordinador que controla el diálogo entre computadoras.

Se encarga de la sincronización y del control de diálogo (quién habla y cuándo).

Ejemplo: Establecimiento de sesiones en aplicaciones como NetBIOS, RPC.

Capa 4: Transporte

Esta capa se encarga de la transferencia confiable de datos entre sistemas. Divide los datos en segmentos y asegura que lleguen correctamente a su destino.

Controla el flujo de datos, la corrección de errores y la retransmisión si es necesario.

Ejemplo: TCP (Transmission Control Protocol) para conexiones confiables, UDP (User Datagram Protocol) para conexiones no confiables.

Capa 3: Red

Esta capa determina la mejor ruta para enviar los datos a través de la red. Se encarga del direccionamiento lógico y del enrutamiento de paquetes.

Divide los segmentos en paquetes y los dirige a través de diferentes redes si es necesario.

Ejemplo: Protocolo IP (Internet Protocol), routers.

Capa 2: Enlace de Datos

Esta capa se encarga de la transferencia confiable de datos a través del medio físico. Toma los paquetes de la capa de red y los convierte en tramas.

Proporciona detección y corrección de errores, y control de acceso al medio.

Ejemplo: Ethernet, PPP (Point-to-Point Protocol), switches.

Capa 1: Física

Esta es la capa más básica, que se encarga de la transmisión física de bits a través del medio de comunicación.

Define las características eléctricas, mecánicas, funcionales y procedimentales para activar, mantener y desactivar el enlace físico.

Ejemplo: Cables de red, conectores, hubs, señales eléctricas u ópticas.
