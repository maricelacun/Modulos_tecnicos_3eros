Modelo TCP/IP

El protocolo que impulsa Internet

El modelo TCP/IP es el conjunto de protocolos de comunicación que permite la interconexión de redes y es la base de Internet. A diferencia del modelo OSI, TCP/IP tiene solo 4 capas y se centra en la funcionalidad práctica más que en la estandarización teórica.

Modelo OSI (7 capas)

Modelo teórico de referencia que define 7 capas para la comunicación entre sistemas.

  • Aplicación
  • Presentación
  • Sesión
  • Transporte
  • Red
  • Enlace de datos
  • Física

Modelo TCP/IP (4 capas)

Modelo práctico utilizado en Internet que combina algunas funciones del modelo OSI.

  • Aplicación
  • Transporte
  • Internet
  • Acceso a la red

Capa 4: Aplicación

Esta capa combina las funciones de las capas de Aplicación, Presentación y Sesión del modelo OSI. Proporciona servicios de red a las aplicaciones de usuario y define los protocolos para la comunicación entre aplicaciones.

Es la interfaz entre el usuario y la red, donde se inician las solicitudes de comunicación.

Protocolos: HTTP, HTTPS, FTP, SMTP, DNS, SSH, Telnet.

Protocolos principales de la capa de Aplicación:

HTTP/HTTPS

Protocolo de transferencia de hipertexto (seguro). Utilizado para navegar por la web.

FTP

Protocolo de transferencia de archivos. Permite subir y descargar archivos.

SMTP

Protocolo de transferencia de correo simple. Utilizado para enviar correos electrónicos.

DNS

Sistema de nombres de dominio. Traduce nombres de dominio a direcciones IP.

Capa 3: Transporte

Esta capa es responsable de la comunicación extremo a extremo entre dispositivos. Proporciona un transporte confiable de datos y control de flujo.

Divide los datos en segmentos (TCP) o datagramas (UDP) y asegura que lleguen correctamente a su destino.

Protocolos: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

Protocolos principales de la capa de Transporte:

TCP

Protocolo orientado a conexión que garantiza la entrega confiable de datos.

UDP

Protocolo no orientado a conexión, más rápido pero menos confiable que TCP.

Capa 2: Internet

Esta capa es responsable del direccionamiento lógico, enrutamiento y entrega de paquetes a través de redes. Es el núcleo del modelo TCP/IP.

Toma los segmentos de la capa de transporte y los convierte en paquetes, añadiendo direcciones IP de origen y destino.

Protocolos: IP (Internet Protocol), ICMP, ARP, IGMP.

Protocolos principales de la capa de Internet:

IP

Protocolo de Internet que proporciona direccionamiento y enrutamiento de paquetes.

ICMP

Protocolo de mensajes de control de Internet. Utilizado para diagnóstico y errores.

ARP

Protocolo de resolución de direcciones. Asocia direcciones IP con direcciones MAC.

Capa 1: Acceso a la Red

Esta capa combina las funciones de las capas Física y Enlace de Datos del modelo OSI. Se encarga de la transmisión física de datos a través del medio de red.

Define cómo los datos se envían físicamente a través de la red, incluyendo el direccionamiento físico (MAC).

Protocolos y tecnologías: Ethernet, Wi-Fi, PPP, Frame Relay, ATM.

Tecnologías principales de la capa de Acceso a la Red:

Ethernet

Tecnología de red de área local (LAN) más común, utiliza cables de cobre o fibra.

Wi-Fi

Tecnología de red inalámbrica que permite la conexión sin cables.

PPP

Protocolo punto a punto. Utilizado para conexiones directas entre dos nodos.
